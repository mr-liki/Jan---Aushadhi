⚠️  NEVER commit API keys to GitHub!

Your .gitignore already protects:
  - local.properties
  - build/ directory
  - .gradle/ directory

To keep API keys safe:

Option A: Use local.properties (Recommended)
  1. Create app/local.properties (NOT committed)
  2. Add your keys:
     MAPS_API_KEY=YOUR_ACTUAL_KEY
     GEMINI_API_KEY=YOUR_ACTUAL_KEY

Option B: Use Environment Variables
  export MAPS_API_KEY="YOUR_KEY"
  export GEMINI_API_KEY="YOUR_KEY"

Option C: Create secrets file
  Create app/secrets.properties (add to .gitignore)
  Add your keys there

================================================================================
                            TROUBLESHOOTING
================================================================================

Issue: "fatal: remote origin already exists"
Solution:
  git remote remove origin
  git remote add origin https://github.com/YOUR_USERNAME/Jan-Aushadhi-Finder.git

Issue: "Permission denied (publickey)"
Solution: Set up SSH keys
  https://docs.github.com/en/authentication/connecting-to-github-with-ssh

Issue: "fatal: 'origin' does not appear to be a 'git' repository"
Solution:
  git remote -v  # Check remotes
  git remote add origin https://github.com/YOUR_USERNAME/Jan-Aushadhi-Finder.git

Issue: Large files rejected
Solution:
  Check .gitignore is working
  Remove build/ and .gradle/ if accidentally committed
  Use: git rm --cached <file>
